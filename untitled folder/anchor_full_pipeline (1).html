Continuous monitoring
Runs every 5 seconds in the background. User doesn't see or feel this.
Window watcher reads the screen
Every 5 seconds, a Python script reads the title of the currently active window on the user's computer. Not a screenshot, not screen recording, not what they're typing -- just the window title. For example: "Google Chrome - Reddit" or "VS Code - server.py" or "Preview - research_paper.pdf". This is the ONLY data collected from the user's screen.
macOS: osascript reads "Google Chrome - Kidney-Friendly Recipes | DaVita"
Windows: pygetwindow reads the same kind of title

Activity tracker monitors keyboard and mouse
A background listener (pynput) records the timestamp of the last keyboard press, mouse movement, or scroll event. It does NOT record what keys were pressed or where the mouse clicked -- only WHEN activity happened. This is used to detect "silent drift" -- when the correct window is open but the user has zoned out, picked up their phone, or left their desk.
last_activity_time = 14:32:47 (last mouse move)
Now = 14:37:12 → idle for 4.4 minutes → trigger silent drift check

Window change detection
The system compares the current window title to the previous one. If they're different, the window changed and we move to the classifier (Phase 3). If they're the same, we check for timeouts instead: Has the user been idle too long? Have they been on Claude/ChatGPT for 15+ minutes? Have they been focused 40+ minutes without a break? Have they never opened a task-relevant app? Is the user stuck on a drift app for 1+ minute?
Previous: "Chrome - Volunteer Application"
Current: "Chrome - Reddit"
→ Window changed! Go to classifier.
Notification pull detection
When a window changes, the system measures how fast the switch happened relative to the user's last keyboard/mouse activity. If the switch was very fast (under 3 seconds since last activity), it was probably a notification popping up (WhatsApp call, Slack message, calendar alert) -- NOT the user deliberately navigating. This distinction matters because notification pulls get a gentler response: "Looks like Slack pulled you away" instead of "You drifted to Slack."
User was typing at 14:32:46
WhatsApp appeared at 14:32:47 (1 second later)
→ NOTIFICATION PULL (user didn't choose to open WhatsApp)
3
Intelligent classification
The LLM decides if each window is related to the task. Content-aware, not app-name-aware.
Content-aware classification
The classifier sends the task context + the window title to Gemini and asks: "Is this window relevant to the user's task?" The critical difference from other apps: it reads the CONTENT in the title, not just the app name. The same app gets different verdicts depending on what the user is doing in it. Messaging apps (WhatsApp, Telegram) always default to "unsure" -- the system asks instead of assuming.
Task: "volunteer application"

"Claude - personal skills assessment" → RELEVANT (content matches)
"Claude - AI agent idea for yhack" → DRIFT (unrelated content)
"YouTube - MIT volunteering lecture" → RELEVANT
"YouTube - best headphones 2026" → DRIFT
"WhatsApp" → UNSURE (could be either)

Result caching
Every classification result is cached. If the user opens Reddit, the classifier calls Gemini (~1.5 seconds). If they go back to Reddit later, the cached result is returned instantly (0ms). This saves money (fewer API calls) and reduces latency. The cache resets at the start of each session because a different task means different relevance judgments.
First time "Chrome - Reddit": 1,431ms (Gemini call)
Second time "Chrome - Reddit": 0ms (cache hit)
Classification result determines what happens next:
RELEVANT Save to observation history silently. No agent call. No LLM cost. User is on task, don't interrupt.
DRIFT Save to history AND call the Anchor Agent. The agent reads the full session context and decides how to respond.
UNSURE Save to history AND call the Anchor Agent. The agent typically asks the user: "Is this for your task or did you drift?"
4
Anchor Agent (the brain)
An ADHD specialist AI that reasons about the situation and chooses the right intervention.
Agent reads full session history
The agent doesn't just see the current event. It reads the entire session history: every window change, every classification, every nudge it already sent, every user response. It also reads session state: how many drifts total, how long the session has been running, whether the user ever started the task, what notifications they're expecting. This is why the same drift event (opening Reddit) gets completely different responses depending on context.
[0m] Session started: "volunteer application"
[0.5m] Window: Application Form → relevant
[5m] Window: Claude - skills help → relevant
[15m] Window: Reddit → drift (1st)
[15.5m] Window: Application Form → relevant (self-corrected)
[20m] Window: Reddit → drift (2nd) ← CURRENT EVENT

Agent sees: "She was focused 20 min, self-corrected once, now drifting again. Possible executive function depletion."
Agent diagnoses WHY the user is drifting
Using its knowledge of ADHD neuroscience (Barkley's executive function fuel tank model, time blindness, task initiation paralysis), the agent diagnoses the root cause of the drift. Different causes get different interventions. This is what makes Anchor an ADHD specialist, not a generic reminder app.
DEPLETED
Was focused 20+ min then suddenly started drifting. Executive function fuel tank is empty. Need to refuel with a break.
STUCK
Was working, hit a specific hard part, started drifting. Need task chunking -- break the hard part into smaller pieces.
OVERWHELMED
Never started the task. Keeps bouncing between random apps. Task feels too big. Need the smallest possible first step.
BORED
Drifting to entertaining apps (Reddit, YouTube). The task is tedious. Need acknowledgment + mini-goal to create momentum.
AVOIDING
Keeps returning to task but drifts again within 1-2 min. Something about the task feels emotionally hard. Need to name it and offer to skip to an easier section.
NOTIFICATION PULL
App popped up on its own (call, message, alert). User didn't choose to drift. Need gentle acknowledgment, not a redirect.
Agent selects tools autonomously
The agent has 6 tools it can choose from. It picks the right tool(s) based on its diagnosis. Sometimes it uses one tool, sometimes it chains multiple. The agent decides autonomously -- we don't hardcode which tool to use for which situation. This is what makes it a real AI agent, not a script.
speak_to_user
Speak a nudge, observation, or encouragement out loud via ElevenLabs voice.
ask_user
Ask the user a question via voice. User speaks back through Agora audio channel.
suggest_break
Suggest a 3-5 min break with a physical activity (stretch, walk, water. Never phone).
chunk_task
Break the task into the smallest possible next step. "Just fill in the first field."
search_adhd_strategy
Search Tavily for research-backed ADHD strategies for this specific situation.
suggest_dnd
Suggest enabling Do Not Disturb when notifications keep pulling the user away.

Progressive cooldown (anti-nagging)
ADHD research shows that nagging makes things worse -- it creates shame and rebellion, not compliance. So the cooldown between nudges gets longer each time. If repeated nudges aren't working, the agent switches strategy entirely (suggests a break instead of another redirect). Eventually it goes quiet and waits for the user to come back on their own.

Drift escalation
The agent's response escalates based on total drifts in the session. But the response is NEVER the same -- it varies its wording, tone, and approach every time. The same situation gets different handling based on the full session story.

Background timeout triggers
These fire from the monitoring loop even when the window doesn't change.
Silent drift detection
The correct window is open but no keyboard or mouse activity for too long. The threshold adapts to the task type: 1 minute for forms/writing/browsing (constant interaction expected), 3 minutes for coding (sometimes you think before typing), 7 minutes for reading (legitimately staring at screen). The agent asks gently: "Your screen has been quiet for a while. Still with me?"
Task initiation detection
1+ minute has passed and the user has NEVER opened a task-relevant app. They declared "volunteer application" but they're bouncing between Terminal and random Chrome tabs. This is task initiation paralysis -- one of the top 3 ADHD executive function deficits. The agent helps them start: "Just open the form and fill in your name. Nothing else."
Sustained drift detection
User opened a drift app (Reddit) and the agent stayed silent (first drift, giving them a chance to self-correct). But they're STILL on Reddit 1+ minute later. They didn't self-correct -- they got sucked in. The agent intervenes even though it was technically the first drift.
Hyperfocus detection
User has been focused for 40+ minutes without any break. This is hyperfocus -- the opposite problem from drift. ADHD brains can lock onto a task for hours, skipping meals, missing meetings, depleting all executive function for the rest of the day. The agent proactively suggests a break before they crash.
Long stay on relevant apps
User has been on Claude/ChatGPT/YouTube for 15+ minutes. These were classified as "relevant" but the user might have drifted WITHIN the app -- started on a work-related Claude conversation and ended up discussing something completely different. The agent checks in: "You've been on Claude for a while. Still working on the application?"

Data sent to LLM: task description + window title. Nothing else.
